## Two reference defects to correct, and a third found during planning

**D-06 (locked):** the reference greps `^##[[:space:]]+Current Phase`
(`multi-ai-review-gate.sh:96`). No STATE.md in the fleet uses that heading —
every one writes `## Current Position`. Match `## Current Position`; tolerate
`## Current Phase` as a fallback.

**D-07 (locked):** the reference's `gsd-tools.cjs` node step
(`multi-ai-review-gate.sh:105-111`) has no Codex analogue —
`~/.codex/get-shit-done/` ships no `bin/`. Omit the step entirely. The resolver
is 4 steps, not 5. This makes D-06 load-bearing: STATE.md is Codex's only
workflow-state source, where on Claude the node step masks the bug.

**Third defect, found while planning this phase — fix it as part of D-06.**
The reference's line regex is `/[Pp]hase[[:space:]]+[0-9]+(\.[0-9]+)?/`. It
**cannot match the canonical `Phase: 08` line**, because `[[:space:]]+` must
match the character right after `hase`, and that character is a colon.
Verified by execution:

  $ echo 'Phase: 08 (Plan-Review Gate) — DISCUSSING' \
      | awk '{ if (match($0, /[Pp]hase[[:space:]]+[0-9]+/)) print "MATCH"; else print "NO MATCH" }'
  NO MATCH

So even with D-06's heading fixed, awk scans past the canonical line and
matches the **first incidental prose mention** of a phase number under the
heading. Verified against two real files:

  - this repo → matches line 35, `"...; phase 08 context gathering started"`.
    Coincidentally yields the right answer (08).
  - `../claude-workflow/.planning/STATE.md` → matches line 48,
    `"Phase 24 (most recent shipped) is at commit ..."` — resolves phase 24,
    which the line itself says is the *most recent shipped*, not the active one.

Correct it: tolerate an optional colon, and anchor on the canonical `Phase:`
line rather than free prose. A prose decoy under the heading MUST NOT win over
the canonical line. Task 1 carries a regression test for exactly this.

This is not re-litigating D-06 — D-06 fixes the heading and says step 2 must
actually work; the line regex is the rest of that same fix. Add it to the
deferred upstream bug report (08-CONTEXT.md "Deferred Ideas"), which currently
names only the heading half.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo filesystem → verifier | `.planning/current-phase` (a symlink), `STATE.md` (parsed text), and `.planning/phases/**` globs are all repo content. ## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | `resolve_phase` step 1, `readlink .planning/current-phase` | mitigate | Canonicalize the resolved pointer and require containment inside `.planning/phases/`; reject and fall through to step 2 otherwise. The reference has no containment check — a crafted `current-phase -> /` or `-> ../../..` would make it stat outside the tree. Task 1 carries the escape case; Task 2 implements the check. |
| T-08-02 | Tampering | `_match_phase_dir`, STATE.md-derived phase number | mitigate | The awk extraction constrains the value to `[0-9]+(\.[0-9]+)?`; `_match_phase_dir` re-asserts that constraint and returns empty otherwise. | T-08-03 | Denial of service | `resolve_phase`, `find` over `.planning/phases` | accept | `-maxdepth 2` bounds the walk. A pathological tree could slow it, but the verifier is operator-invoked on a local repo the operator already trusts enough to open in an editor; there is no privilege gain. |
| T-08-04 | Information disclosure | verifier reads | accept | The verifier reads only `.planning/`-rooted files in the cwd and writes only to stderr. | T-08-05 | Elevation of privilege | `set -e` abort mid-resolution | mitigate | `set -uo pipefail` (not `set -e`) plus deliberate exit statements: a gate that dies mid-resolution exits non-zero-non-2 and would read as neither a clean allow nor a block. Every terminal path is an explicit `exit 0` or `exit 2`. |
| T-08-SC | Tampering | supply chain / package installs | accept | This plan installs zero packages — it ships one bash script and edits one bash harness. No `npm`/`pip`/`cargo` surface exists, so the Package Legitimacy Gate does not apply and no `[ASSUMED]`/`[SUS]` checkpoint is warranted. |
