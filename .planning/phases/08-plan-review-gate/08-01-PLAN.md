**`set -u` discipline — mandatory, stated here because both `08-01` and `08-02`
share this file's `set -uo pipefail` header.** Under `set -u`, an unguarded read
of an unset variable aborts the script mid-resolution, and an abort is neither a
clean allow nor a block — it is the T-08-05 silent-bypass failure mode this
verifier exists to prevent. Therefore **every** environment and positional read
MUST use the default-expansion form:

  ${GSD_SKIP_REVIEWS:-}     ${GSD_PLAN_REVIEW_DEBUG:-}     ${CODEX_HOME:-$HOME/.codex}
  ${1:-}                    ${2:-}                          ${REPO_ROOT:-}

A single bare `$1` in argv parsing is a defect. This applies to plan `08-02`'s
additions to the same file.

**Portable path canonicalization — pin this idiom, do not improvise.**
`realpath -m` does not ship on stock macOS and `readlink -f` differs between BSD
and GNU. The portable idiom for a directory is a subshell `cd` + `pwd -P`:

  _canon_dir() { ( cd "${1:-}" 2>/dev/null && pwd -P ); }

It resolves symlinks and `..` traversal, prints nothing on a non-existent or
unreadable path, and is available everywhere. Both Unreleased migration bugs in
this repo's CHANGELOG were relative-path resolution failures, so this surface is
opinionated by design rather than left to implementor taste.

**Portable mtime — pin this helper too, same reason.** (08-REVIEWS.md round 2,
Codex, LOW: the plan required "portable mtime comparison" without pinning the
fallback, leaving two implementers free to diverge.) `stat` is the classic
BSD/GNU split: GNU wants `stat -c %Y`, BSD/macOS wants `stat -f %m`, and neither
flag exists on the other. Pin the try-GNU-then-BSD shape:

  _mtime() { stat -c %Y "${1:-}" 2>/dev/null || stat -f %m "${1:-}" 2>/dev/null; }

It prints an integer epoch or nothing. Callers MUST treat empty as "unknown" and
skip the candidate rather than comparing an empty string numerically (which
aborts under `set -u`-adjacent arithmetic). Do not reach for `find -printf`
(GNU-only) or `ls --time-style` (GNU-only).

**The mtime loop must NOT run in a subshell.** A `find ... | while read` pipeline
puts the loop body in a subshell on bash, so the winner variable it assigns is
discarded at the pipe's close and the loop's selected value never escapes — the
selection silently yields empty and the resolver falls through to fail-open. Use
process substitution so the loop runs in the current shell:

  while IFS= read -r -d '' f; do ...; done < <(find .planning/phases -maxdepth 2 -name '*-PLAN.md' -print0)

This is a correctness requirement, not a style preference: with the pipeline
shape, Task 1's "newer wins" case fails and the whole step is dead code.

<root_location>
## The verifier locates the repo root itself — corrected requirement

An earlier draft of this plan specified "Runs from the repo root (the caller's
cwd). Resolves paths relative to cwd." Cross-AI review (08-REVIEWS.md, Codex,
HIGH) identified this as a silent fail-open: an agent editing `src/app.ts` and
invoking the verifier from `src/` sees no `.planning/`, resolves nothing, and
the gate exits 0. That is an ordinary invocation, not an exotic one, and the
consequence is that the gate authorizes the edit it exists to block.

**Requirement.** Before any resolution work, establish `REPO_ROOT` and `cd` to
it (or prefix every path with it) for the remainder of the script:

1. `git rev-parse --show-toplevel` — the primary mechanism. Guard it: it exits
   non-zero outside a git tree and prints to stderr; suppress both.
2. If that yields nothing (not a git tree, or `git` absent): walk upward from
   `$PWD` looking for the nearest ancestor directory containing a `.planning`
   directory. Stop at `/`. This covers a non-git checkout and a worktree the
   operator exported.
3. If neither yields a root: fail open (exit 0) with a `GSD_PLAN_REVIEW_DEBUG`
   diagnostic. There is no planning tree to enforce against.

Once `REPO_ROOT` is established, `cd "$REPO_ROOT"` so the four resolution steps
and every grandfather guard operate against a known root rather than the
caller's incidental cwd. Emit the resolved root under
`GSD_PLAN_REVIEW_DEBUG=1` as `repo-root: <dir>`.

This requirement sits under 08-CONTEXT.md's "Claude's Discretion" for verifier
path resolution. It does not conflict with any locked decision.

The four items below are **resolver requirements this port must satisfy**, added
after cross-AI review (08-REVIEWS.md). They are not classified as reference
defects and **must not be added to ADR-0009's "three reference defects"
enumeration** — they are gaps in this plan's own prior specification, closed
here.

**4. STATE.md parsing MUST be bounded to the Current Position section.**
(08-REVIEWS.md, Codex, MEDIUM.) Setting a flag on the heading and never clearing
it lets an anchored `Phase:` line in a *later* H2 — `## Notes`, `## History` —
win. Clear the flag on the next `^##` heading. Parsing ends at the section
boundary, not at EOF.

**5. Decimal inserted phases MUST zero-pad the integer part.**
(08-REVIEWS.md, Codex, HIGH.) ROADMAP.md §"Phase Numbering" documents decimal
phases (`8.1`, `8.2`) as supported for urgent insertions, and the phase-directory
layout is zero-padded (`08.1-<slug>`). The reference's padding rule retries a
zero-padded glob only when the whole value is a single digit, so `8.1` searches
`8.1-*` and finds nothing. Pad the **integer part**: `8.1` → try `8.1-*`, then
`08.1-*`. `8` → `8-*`, then `08-*`. `12.3` → `12.3-*` (no pad needed, the integer
part is already two digits). `08.1` → `08.1-*` (already padded).

**6. Ambiguous phase-directory matches MUST be a TERMINAL fail-open, via an
explicit tri-state contract.** (08-REVIEWS.md round 1, Codex, MEDIUM; **hardened
round 2, Codex, HIGH.**) If both `08-old/` and `08-plan-review-gate/` exist,
`find` returns whichever it encounters first and the resolver silently gates
against a directory the operator did not mean.

Round 1's fix — "count the matches; two or more → return empty and print a
diagnostic" — is **not sufficient, and round 2 caught why.** Returning *empty*
for ambiguity makes it indistinguishable from "no match". `resolve_phase` reads
empty as "this step didn't resolve, continue" and falls through to step 3
(newest-plan-by-mtime), which happily picks one of the two ambiguous directories
by timestamp — the exact arbitrary selection the diagnostic just warned about.
The diagnostic prints, and then the resolver does the wrong thing anyway.

**The contract is tri-state. Signal it with the exit status, not the stdout:**

  0 = unique   → the single match is echoed on stdout; caller uses it
  1 = absent   → nothing echoed; caller CONTINUES to the next resolution step
  2 = ambiguous → nothing echoed; caller STOPS. Terminal fail-open.

Status 2 is terminal: `resolve_phase` must return immediately with no phase, the
verifier exits 0, and **no later step runs**. An operator with two directories
claiming one number has a broken tree; guessing which one to gate against is
worse than declining. The diagnostic naming every match prints to stderr
unconditionally — not only under `GSD_PLAN_REVIEW_DEBUG` — because a fail-open
the operator cannot see is the ADR-0018 drift pattern; this one announces itself.

**Do not distinguish the states by stdout emptiness.** Both 1 and 2 echo
nothing; only `$?` separates them. A caller that branches on `[ -z "$dir" ]`
instead of on the status has re-introduced the bug.

**7. The newest-plan-by-mtime step MUST be portable, NUL-safe, and
deterministic.** (08-REVIEWS.md, Codex, MEDIUM.) The reference's
`find -print0 | xargs -0 ls -t | head -1` is not safe to port as-is: `ls -t`
re-emits newline-delimited names (so the `-print0` NUL-safety is discarded at
the boundary), BSD and GNU `xargs` disagree on empty input (GNU runs the command
once with no args; BSD may not), and `ls -t` order among equal mtimes is
unspecified. Replace it with an explicit, deterministic selection: iterate the
`find -print0` results with a `while IFS= read -r -d ''` loop **fed by process
substitution, not a pipe** (`<interfaces>` — a pipe puts the loop in a subshell
and the selected value is discarded), track the best candidate by comparing
mtimes obtained via the pinned `_mtime` helper (`<interfaces>` — the BSD/GNU
`stat` split is pinned there, not left to taste), and break ties by lexical path
order so the result is stable. A candidate whose `_mtime` returns empty is
skipped, never compared as an empty string. Empty input must yield empty output
and must not invoke a downstream command at all. Task 1 carries an equal-mtime
case and an empty-input case.
</resolver_defects>
