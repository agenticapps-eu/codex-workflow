<ordering>
## Guard order is load-bearing — get this right

The reference (`multi-ai-review-gate.sh`) orders its guards for a PreToolUse
hook with a stdin payload. Ours has a different shape. The required order:

1. `GSD_SKIP_REVIEWS=1` → allow. Before any filesystem work. (Reference line 46.)
2. `--file` bypass list, only when `--file` was supplied → allow.
   (Reference lines 52-60.)
3. Resolve the phase (plan `08-01`, after locating the repo root). Empty/non-dir
   → allow.
4. `multi-ai-review-skipped` marker, checked **ONLY at the resolved phase dir**,
   which plan `08-01`'s containment check has already validated → allow.
5. Grandfather guards: no plans → allow; `*-SUMMARY.md` → allow; legacy bare
   `PLAN.md` → allow. (Plan `08-01`.)
6. Collect `*-REVIEWS.md` under the resolved phase at `-maxdepth 2`.
   Zero matches → **exit 2**. Two or more matches → **exit 2** (ambiguous).
7. The single match is not a regular file (FIFO, socket, directory, dangling
   symlink) → **exit 2**, treated as MISSING. Guard precedes any read.
8. Frontmatter present and well-formed → count DISTINCT normalized `reviewers:`
   entries; `< 2` → **exit 2**. Then check `plans_reviewed:` covers every current
   `*-PLAN.md`; a gap → **exit 2**.
9. Frontmatter present but MALFORMED (opening `---` with no closing `---`) →
   **exit 2**. Do not fall through to step 10.
10. Frontmatter absent entirely → `< 5` lines → **exit 2**; else allow.

Step 10 is where this host deliberately diverges from the reference. The
reference warns and exits 0 (its lines 166-172). D-13 requires exit 2. This is
the single most load-bearing behavioral change from the reference — do not
"fix" it back toward the reference's shape.

## Three corrections review forced into this order — do not restore the old shape

**Step 4 no longer checks `.planning/current-phase/multi-ai-review-skipped`
directly.** (08-REVIEWS.md, Codex, HIGH.) The reference checks both that raw path
and the resolved dir (its lines 130-131). But plan `08-01` rejects a
`current-phase` pointer that escapes `.planning/phases/` — and a raw check on
`.planning/current-phase/` **follows the very symlink the containment check just
rejected**, letting a marker outside the planning tree authorize the edit. When
the pointer IS legitimate, `.planning/current-phase/multi-ai-review-skipped` and
`<resolved>/multi-ai-review-skipped` are the same file, so checking only the
resolved path loses nothing legitimate and closes the escape. This tightens the
port; it does not touch D-11, which requires "a per-phase marker file" — the
resolved phase dir is that per-phase location.

**Step 7 blocks instead of allowing.** (08-REVIEWS.md — Codex HIGH says a FIFO
`08-REVIEWS.md` returning exit 0 is a trivial gate bypass; OpenCode read the same
threat as adequately mitigated. Resolved toward fail-closed by operator
decision.) An earlier draft of this plan said "not a regular file → allow (a
FIFO/socket would hang `wc -l`)". Avoiding the hang is correct; **authorizing the
edit is not.** A non-regular review artifact is not evidence of a review — it is
either an accident or an attack, and in both cases the honest verdict is MISSING.
Block with exit 2 and a message naming the non-regular path.

**Step 6 requires exactly one artifact.** (08-REVIEWS.md, Codex, HIGH.) An
earlier draft used `find ... '*-REVIEWS.md' | head -1`, which silently accepts
whichever file `find` returns first — a nested or stale artifact can win over the
canonical one. Count the matches and block on ambiguity, mirroring plan `08-01`'s
`_match_phase_dir` treatment of ambiguous phase dirs.
</ordering>
