## Guard order is load-bearing — get this right

The reference (`multi-ai-review-gate.sh`) orders its guards for a PreToolUse
hook with a stdin payload. Ours has a different shape. The required order:

1. `GSD_SKIP_REVIEWS=1` → allow. Before any filesystem work. (Reference line 46.)
2. `--file` bypass list, only when `--file` was supplied → allow.
   (Reference lines 52-60.)
3. Resolve the phase (plan `08-01`). Empty/non-dir → allow.
4. `multi-ai-review-skipped` marker, checked at BOTH `.planning/current-phase/`
   and the resolved phase dir → allow. (Reference lines 130-131. It must come
   after resolution because one of the two locations *is* the resolved dir.)
5. Grandfather guards: no plans → allow; `*-SUMMARY.md` → allow; legacy bare
   `PLAN.md` → allow. (Plan `08-01`.)
6. REVIEWS.md absent → **exit 2**.
7. REVIEWS.md not a regular file → allow (a FIFO/socket would hang `wc -l` —
   reference line 162). This check MUST precede any read of the file, not follow
   the absent-check as it does in the reference.
8. Frontmatter present → count `reviewers:`; `< 2` → **exit 2**.
9. Frontmatter absent → `< 5` lines → **exit 2**; else allow.

Step 9 is where this host deliberately diverges from the reference. The
reference warns and exits 0 (its lines 166-172). D-13 requires exit 2. This is
the single most load-bearing behavioral change from the reference — do not
"fix" it back toward the reference's shape.
</ordering>
