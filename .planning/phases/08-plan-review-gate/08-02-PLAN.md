## Guard order is load-bearing — get this right

The reference (`multi-ai-review-gate.sh`) orders its guards for a PreToolUse
hook with a stdin payload. Ours has a different shape. The required order:

1. `GSD_SKIP_REVIEWS=1` → allow. Before any filesystem work. (Reference line 46.)
2. `--file` bypass list, only when `--file` was supplied → allow.
   (Reference lines 52-60.)
3. Resolve the phase (plan `08-01`, after locating the repo root). Empty/non-dir
   → allow.
4. `multi-ai-review-skipped` marker, checked **ONLY at the resolved phase dir**,
   which plan `08-01`'s containment check has already validated → allow.
5. Grandfather guards: no plans → allow; `*-SUMMARY.md` → allow; legacy bare
   `PLAN.md` → allow. (Plan `08-01`.)
6. Collect `*-REVIEWS.md` under the resolved phase at `-maxdepth 2`.
   Zero matches → **exit 2**. Two or more matches → **exit 2** (ambiguous).
7. The single match is not a regular file (FIFO, socket, directory, dangling
   symlink) → **exit 2**, treated as MISSING. Guard precedes any read.
7a. The single match IS a symlink (live or dangling) → **exit 2**. `[ -f ]` alone
   does NOT close this: it follows symlinks, so `-f` is TRUE for a live symlink
   pointing at any regular file anywhere. Test `[ -L ]` FIRST and reject.
   (08-REVIEWS.md round 2, Codex, HIGH.)
8. Frontmatter present and well-formed → count DISTINCT normalized `reviewers:`
   entries; `< 2` → **exit 2**. Then check `plans_reviewed:` covers every current
   `*-PLAN.md`; a gap → **exit 2**.
9. Frontmatter present but MALFORMED (opening `---` with no closing `---`) →
   **exit 2**. Do not fall through to step 10.
10. Frontmatter absent entirely → `< 5` lines → **exit 2**; else allow.

Step 10 is where this host deliberately diverges from the reference. The
reference warns and exits 0 (its lines 166-172). D-13 requires exit 2. This is
the single most load-bearing behavioral change from the reference — do not
"fix" it back toward the reference's shape.

## Three corrections review forced into this order — do not restore the old shape

**Step 4 no longer checks `.planning/current-phase/multi-ai-review-skipped`
directly.** (08-REVIEWS.md, Codex, HIGH.) The reference checks both that raw path
and the resolved dir (its lines 130-131). But plan `08-01` rejects a
`current-phase` pointer that escapes `.planning/phases/` — and a raw check on
`.planning/current-phase/` **follows the very symlink the containment check just
rejected**, letting a marker outside the planning tree authorize the edit. When
the pointer IS legitimate, `.planning/current-phase/multi-ai-review-skipped` and
`<resolved>/multi-ai-review-skipped` are the same file, so checking only the
resolved path loses nothing legitimate and closes the escape. This tightens the
port; it does not touch D-11, which requires "a per-phase marker file" — the
resolved phase dir is that per-phase location.

**Step 7 blocks instead of allowing.** (08-REVIEWS.md — Codex HIGH says a FIFO
`08-REVIEWS.md` returning exit 0 is a trivial gate bypass; OpenCode read the same
threat as adequately mitigated. Resolved toward fail-closed by operator
decision.) An earlier draft of this plan said "not a regular file → allow (a
FIFO/socket would hang `wc -l`)". Avoiding the hang is correct; **authorizing the
edit is not.** A non-regular review artifact is not evidence of a review — it is
either an accident or an attack, and in both cases the honest verdict is MISSING.
Block with exit 2 and a message naming the non-regular path.

**Step 6 requires exactly one artifact.** (08-REVIEWS.md, Codex, HIGH.) An
earlier draft used `find ... '*-REVIEWS.md' | head -1`, which silently accepts
whichever file `find` returns first — a nested or stale artifact can win over the
canonical one. Count the matches and block on ambiguity, mirroring plan `08-01`'s
`_match_phase_dir` treatment of ambiguous phase dirs.

## Two bypasses round 2 found — both are authorization holes, close both

Round 1 hardened the non-regular-file case toward fail-closed. Round 2 found that
the resulting guard **does not actually close what it claims to**, and found a
second hole beside it (08-REVIEWS.md round 2, Codex, HIGH ×2). Both are gate
bypasses: each lets an edit through that the gate exists to block.

**Bypass 1 — `[ -f "$REVIEWS" ]` follows symlinks.** The round-1 correction says
"require a regular file before any read; `-f` is false for a FIFO, socket,
directory, and dangling symlink". Three of those four are right. But `-f` is a
**dereferencing** test: for a *live* symlink pointing at a regular file, `-f`
returns TRUE. So `ln -s /etc/hosts .planning/phases/08-x/08-REVIEWS.md` is a
five-plus-line frontmatter-less regular file as far as the guard is concerned —
it sails through step 7, has no frontmatter, clears the ≥5-line fallback, and the
gate exits 0. The artifact is not in the phase dir at all, was never a review, and
the operator never wrote it.

Close it **before** the `-f` test, not after:

  - `[ -L "$REVIEWS" ]` → **exit 2**, message naming the path and stating that a
    symlinked review artifact is treated as missing. Test `-L` FIRST: it is the
    only test that does not dereference, so it is the only one that can see the
    symlink at all.
  - Then `[ -f "$REVIEWS" ]` for the remaining non-regular cases (FIFO, socket,
    directory), unchanged from round 1.

Reject the symlink outright rather than canonicalizing-and-containing it. A
containment check would *also* close the escape, but it would admit an
in-phase-dir symlink — and there is no legitimate reason for a review artifact to
be a symlink at all. Rejecting is simpler, has no traversal-normalization edge
cases, and fails closed. Note the asymmetry with plan `08-01`'s `current-phase`
pointer, which IS legitimately a symlink and therefore IS canonicalized-and-
contained: a pointer is *meant* to indirect; an evidence artifact is not.
Comment the reason at the call site so a later reader does not "harmonize" the
two by adding containment here.

**Bypass 2 — the `--file` bypass matches a traversal path textually.** The rule
is "gate on the path prefix AND the basename, never the basename alone" — the
reference's FLAG-A fix. Both halves are textual. So
`.planning/../docs/IMPLEMENTATION-PLAN.md` has the `.planning/` prefix ✓ and the
`PLAN.md` basename ✓ — and it resolves to `docs/IMPLEMENTATION-PLAN.md`, the
exact file the FLAG-A fix exists to keep OUT. The round-1 fix closed the
basename-only hole and left a traversal-shaped hole in the same guard, and the
existing negative case (`--file docs/IMPLEMENTATION-PLAN.md`) does not catch it
because it omits the traversal prefix.

Close it by **rejecting any path with a `..` component before the prefix test
runs**: split the path on `/` and reject if any component is exactly `..`. Reject
rather than normalize-then-test. Normalizing is defensible but requires the path
to exist (the pinned `_canon_dir` idiom `cd`s, and the `--file` target is a file
about to be *created* in the general case), and a `..` in a `--file` argument has
no legitimate use — the caller is naming the file it is about to edit, and no
agent needs traversal to do that. Rejecting is a two-line guard with no
filesystem dependency. A rejected `--file` means the bypass does not fire and the
gate proceeds to normal resolution — it is NOT an error and NOT a block by
itself; the phase's real state then decides.
</ordering>
