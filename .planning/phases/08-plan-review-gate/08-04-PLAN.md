`docs/briefs/plan-review-gate.md` lines 140-151 and confirmed by 08-PATTERNS.md

  "pre_execution": {

`verifier` is a NEW key no other gate carries. That is intentional per D-01's
hybrid decision: every other gate is purely declarative (agent-read); this one
alone carries a programmatic-verifier pointer. ADR-0009 (plan `08-03`) records why.

`post_phase`, `finishing`, each a flat object keyed by gate-slug, each gate
object carrying `skill`, `fires_when`, and gate-specific fields. `pre_execution`
becomes a fifth group, inserted as the first key of `hooks` (trigger-order
readability; JSON key order is not semantically significant to any consumer in
this repo).

**The two config files are near-identical siblings.** `.planning/config.codex.json`
is instantiated from `templates/config-hooks.json`. They differ today ONLY in
`implements_spec` (`0.1.0` in this repo's config vs `0.4.0` in the template) — a
pre-existing drift 08-CONTEXT.md flags as **out of scope**. Do not "fix" it.
