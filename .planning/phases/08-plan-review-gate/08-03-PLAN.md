## What cross-AI review changed in this plan — implement these, they are adjudicated

`08-REVIEWS.md` raised four things this plan owns. The operator adjudicated each.

**1. The egress boundary is advisory, not enforced.** (Codex, HIGH.) Enumerating
approved paths does not constrain an agentic vendor CLI: it can read the whole
working tree, `$HOME`, and tool configuration regardless of which paths the
prompt names. **Adjudication: add a consent gate and document honestly. Do NOT
build a read-only review bundle** — it would not constrain `$HOME` access either,
and it is out of scope for this phase. Task 1 adds affirmative confirmation after
the manifest and before transmission; Task 2's ADR states the boundary is
advisory and records the bundle as considered-and-deferred.

**Evidence to cite as an observation from this repo, not a hypothetical:** during
the review run that produced `08-REVIEWS.md`, the `opencode` reviewer ignored its
prompt and spent roughly ten minutes autonomously reading the repository and
executing `migrations/run-tests.sh` before being re-invoked with tool use
explicitly discouraged. It is recorded in `08-REVIEWS.md`'s own provenance table.

**2. No behavioral producer test.** (Codex HIGH; OpenCode HIGH from the other
side.) Adjudication: the round-trip contract test is **owned by plan `08-02`**
(`test_check_plan_review_contract`). This plan's obligation is to make the
producer's output *extractable and testable* — that is the `reviews-skeleton`
marker pair in `<interfaces>`. Do not add a test here; do add the marker.

**3. Provenance must record provider/model.** (Codex, MEDIUM.) CLI names do not
guarantee vendor diversity — `opencode` is a client, and in this repo's own run it
resolved to `glm-5.2`. Record provider and model in the provenance table.

**4. A hanging reviewer CLI is unhandled.** (Gemini, the one substantive
suggestion; corroborated by the ten-minute `opencode` hang above.) The procedure
covers non-zero exit and empty output but not a hang. Add a per-invocation
timeout; a timeout is recorded as an unavailable reviewer. **Round 2 pinned the
value** (OpenCode, MEDIUM — an unpinned timeout beside a pinned everything-else
invites drift): **300s default, `CODEX_PLAN_REVIEW_TIMEOUT` to override**. See
`<interfaces>` for the calibration. The executor does not choose this number.

Also adopted, from the same review: reviewer independence must be explicit
(each reviewer gets the same bundle, none sees another's output), and verbatim
reviewer text must be fenced under a notice that later agents treat it as data.
Full §14 prompt-injection defense stays deferred (08-CONTEXT.md `<deferred>`) —
this is fencing plus a notice, not a defense.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| **repo → third-party vendor CLIs** | THE major boundary of this phase. `codex-plan-review` passes plan + CONTEXT + canonical-ref file contents to `claude`, `gemini`, `opencode` — external processes that transmit to vendor infrastructure. | Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-12 | Information disclosure | `codex-plan-review` → external vendor CLIs | mitigate (partial — boundary is advisory) | **Revised after review (08-REVIEWS.md, Codex, HIGH: "enumerating approved paths does not constrain tool access").** This is the phase's only egress surface and the reason D-10 forbids the verifier from auto-invoking reviewers. Four controls, all authored into the skill: (1) the skill is **operator-invoked only**; the verifier detects and reports, never invokes. (2) The skill **enumerates the exact file set and vendor list before sending**. (3) **Affirmative operator confirmation is required after the manifest and before any transmission** — consent is never implied by invocation, because the operator may not know which canonical refs the ROADMAP resolves to, and those are the files most likely to surprise. (4) The set is **bounded** to the phase dir + ROADMAP-declared canonical refs, and refuses secret-shaped paths (`.env*`, `*credentials*`, `*.pem`, `*.key`, `.git/`) even when a canonical ref names one. **Honest limit, stated in the skill and in ADR-0009 decision 10:** controls (2) and (4) are advisory. A read-only bundle was considered and deferred: it would not constrain `$HOME` either, so it would buy a stronger claim rather than a stronger control. | T-08-14 | Tampering | plan/CONTEXT content → vendor CLI invocation | mitigate | Repo content flows into a shell command line.
