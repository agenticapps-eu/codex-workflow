|-----------|----------|-----------|-------------|-----------------|
| T-08-12 | Information disclosure | `codex-plan-review` → external vendor CLIs | mitigate | This is the phase's only egress surface and the reason D-10 forbids the verifier from auto-invoking reviewers — plan content must never leave the machine without an explicit operator act. Three controls, all authored into the skill: (1) the skill is **operator-invoked only**; the verifier detects and reports, never invokes. (2) The skill **enumerates the exact file set before sending**, so the operator sees what leaves and to whom. (3) The set is **bounded** to the phase dir + ROADMAP-declared canonical refs, and refuses secret-shaped paths (`.env*`, `*credentials*`, `*.pem`, `*.key`, `.git/`) even when a canonical ref names one. Residual risk — plan text does reach three vendors by design — is the gate's accepted cost, recorded in ADR-0009's Consequences and bounded by the escape hatches for operators who decline it. |
| T-08-13 | Spoofing | reviewer provenance in REVIEWS.md | mitigate | A fabricated reviewer section makes the gate worse than absent: it launders a skipped review into an artifact that satisfies the verifier and that a human reads as evidence. The skill MUST capture each CLI's exit code, record a failed/empty reviewer as **unavailable with its reason**, drop it from `reviewers:`, and re-check the >= 2 minimum — refusing if it now falls short. Never synthesize, paraphrase, or infer a reviewer's output. This is ADR-0002's named risk ("MUST detect this and fall back to a clear message — not silently skip the gate") applied per vendor. Listed as an explicit `## Failure modes` entry. |
| T-08-14 | Tampering | plan/CONTEXT content → vendor CLI invocation | mitigate | Repo content flows into a shell command line. Build the prompt with a **quoted** heredoc (`<<'PROMPT'`), which suppresses shell expansion, so plan text containing backticks, `$(...)`, or quotes cannot be executed by the invoking shell. Prefer passing file paths for the reviewer to read over interpolating file bodies into argv, which also avoids argv length limits. |
| T-08-15 | Tampering | vendor CLI output → REVIEWS.md → later agents | accept | Reviewer output is embedded verbatim (D-12 requires verbatim sections) and is later read by agents, so a compromised or manipulated reviewer could attempt prompt injection at the next reader. Accepted for this phase: §14 prompt-injection defense is explicitly deferred in 08-CONTEXT.md (spec v0.6.0, own phase), and the reviewers are CLIs the operator installed and already trusts with their codebase. ADR-0009's follow-ups name it. |
| T-08-16 | Repudiation | which reviewers actually ran | mitigate | `reviewers:`, `overall_verdict:`, `reviewed_at:` frontmatter plus a prose provenance paragraph naming which CLIs ran and which were unavailable and why — the shape the real dashboard artifact uses, which 08-CONTEXT.md §"Specific Ideas" calls out as worth reproducing. |
| T-08-SC | Tampering | supply chain / package installs | accept | This plan installs zero packages — it authors two markdown documents. No `npm`/`pip`/`cargo` surface exists, so the Package Legitimacy Gate does not apply. The external CLIs (`claude`, `gemini`, `opencode`) are **operator-installed prerequisites detected at runtime**, never installed by this skill; the skill's response to an absent CLI is to refuse (D-14), never to install one. |
</threat_model>
