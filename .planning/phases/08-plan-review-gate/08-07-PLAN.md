The two defect sites in `skills/agentic-apps-workflow/scripts/check-plan-review.sh`:

- **Line 537** — `_cpr_fm_first_line="$(head -n 1 "$REVIEWS" 2>/dev/null || true)"`
- **Line 539** — `if [ "${_cpr_fm_first_line:-}" = "---" ]; then` — the byte-exact
  comparison. This is CR-01. The `else` branch at line 598 is the D-13 >=5-line
  fallback, which performs NO reviewer check.
- **Line 540** — `_cpr_fm_close_line="$(awk 'NR > 1 && $0 == "---" { print NR; exit }' "$REVIEWS")"`
  — the closing-delimiter search, also byte-exact. Must be normalized in the
  same way so open and close agree.
- **Line 545** — `_cpr_fm_block="$(awk -v endline="$_cpr_fm_close_line" 'NR > 1 && NR < endline' "$REVIEWS")"`
  — the extracted block text fed to `_cpr_fm_list`.
- **Lines 550-561** — the `_cpr_reviewers_distinct` pipeline
  (`_cpr_fm_list | sed quote-strip | awk lowercase+trim | sed '/^$/d' | sort -u | wc -l`)
  followed by `if [ "$_cpr_reviewers_distinct" -lt 2 ]; then _cpr_block "found ..."`.
  This is WR-01: it counts distinct strings, never identity.

Helper contracts (already correct — do not modify):
- `_cpr_fm_list <block-text> <key>` — parses BOTH `key: [a, b]` flow style and
  `key:` + `  - a` block style. Bounded to the block text passed in.
- `_cpr_block <reason>` — prints the D-10 block message (phase, missing
  artifact, `Reason: <reason>`, remedy, both escape hatches) to stderr and
  `exit 2`.

Test harness contracts in `migrations/run-tests.sh` (reuse exactly; do NOT
write a second sandbox/exit-code helper):
- `_cpr_enf_phase <sandbox-root> <phase-dir-name> [plan-basename ...]` →
  creates `.planning/phases/<name>/` with one `*-PLAN.md` per basename
  (default `08-01-PLAN.md`), points `.planning/current-phase` at it via a
  `.planning/`-relative symlink, echoes the phase dir path.
- `_cpr_case <label> <sandbox> <expected-exit> [--err-out <path>] [-- <args...>]`
  → runs the verifier in the sandbox, asserts exit code, increments PASS/FAIL.
- `_cpr_check_contains <label> <errfile> <needle>` → literal `grep -qF` on a
  captured stderr file.
- `test_check_plan_review_enforcement()` starts at line 2089; its fixture
  sections are separated by `# ── <name> ───` comment rules. Add new cases
  inside this function.
- Dispatcher: `bash migrations/run-tests.sh check-plan-review` runs the three
  verifier suites only. Bare `bash migrations/run-tests.sh` runs everything.

Verified during planning — the executor can rely on these, they are not guesses:
- BSD awk 20200816 (macOS) `[[:space:]]` DOES match `\r`, so `_cpr_fm_list`'s

  block-extraction time anyway makes correctness independent of awk/locale
  semantics — prefer that over relying on the class definition.
- No existing test pins the "closing delimiter with a trailing space →
  MALFORMED" behavior, so mirroring the normalization onto the closing search
  breaks no existing assertion.
- The real `08-REVIEWS.md` is `reviewers: [gemini, codex, opencode]` → 2
  external after the D-15 exclusion → still exit 0, but with ZERO margin.
  `test_check_plan_review_contract` reads this real artifact and asserts exit 0.
- The `codex-plan-review` skeleton is `reviewers: [gemini, claude, opencode]`
  → unaffected by the exclusion.
- `test_check_plan_review_contract`'s mutated variants use `[gemini]` and
  `[gemini, gemini]` → both already expect exit 2 and are unaffected.
