<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub → CI runner | GitHub Actions executes this repo's own test suite on ephemeral runners. The only code executed is the repo's own `migrations/run-tests.sh` plus the pinned `actions/checkout@v7` action and Ubuntu's `apt` `gawk` package. |
| fork PR → workflow | A `pull_request` from a fork can trigger this workflow. For same-repo PRs the workflow file from the head branch runs; for fork PRs GitHub uses the base-branch workflow and grants a read-only `GITHUB_TOKEN` with no secret access. |
| public submodule → checkout | `vendor/agenticapps-shared` is cloned unauthenticated (public repo). No token/deploy-key crosses this boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Information Disclosure | Secrets/token exposure in workflow logs | accept | The workflow references NO secrets and NO custom `env` secrets; the submodule is public so no `token`/`ssh-key` is passed to checkout. There is nothing sensitive to leak into logs. Documented explicitly rather than assumed. |
| T-10-02 | Elevation of Privilege | `pull_request` trigger running untrusted fork code | accept | The workflow runs only the repo's own test suite; no third-party PR code beyond that suite executes. Because no secrets are used and the default `GITHUB_TOKEN` for fork PRs is read-only, a fork PR cannot exfiltrate anything or escalate. No `pull_request_target` is used (which would be the dangerous variant). |
| T-10-03 | Tampering | Supply chain — `actions/checkout@v7` and `apt-get install gawk` | mitigate | `actions/checkout@v7` is the current major, verified live against the actions/checkout releases API (STACK.md). `gawk` is installed from Ubuntu's trusted base apt repositories, not from npm/pip/cargo — so the package-legitimacy gate (npm/pip/cargo only) does not apply and no legitimacy checkpoint is required. |
| T-10-04 | Denial of Service | Matrix runs on every push and PR | accept | Two-leg matrix on a low-volume internal tooling repo; runner-minute cost is negligible and bounded. No mitigation warranted. |
</threat_model>
