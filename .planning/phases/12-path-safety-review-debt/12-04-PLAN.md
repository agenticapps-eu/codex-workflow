<objective>
Close the single verification gap in Phase 12: 12-01-PLAN.md truth #4 FAILED.
A `--file` value naming a not-yet-created file in a not-yet-created dir under
`.planning/` exit-2-BLOCKS (instead of falling through / accepting) whenever an
unrelated active phase is mid-review (`current-phase` → a phase dir with a
PLAN.md and no REVIEWS.md). The pre-Phase-12 script returned exit 0 for the same
input; Phase 12's canonicalize-and-contain bypass no longer fires on a
non-existent parent (`_canon_dir` returns empty, per D-02), so control falls
through to `resolve_phase`, which resolves the DIFFERENT active phase and blocks.

Purpose: restore the fail-safe accept the milestone promised ("never
exit-2-blocks, never fail-open") WITHOUT reopening the symlink-escape hole WR-03
just closed. The chosen fix (locked upstream, do not re-litigate) is a lexical
`$REPO_ROOT/.planning`-rooted fallback that fires ONLY when `_canon_dir` returns
empty (parent genuinely does not exist). When the parent exists, the existing
resolve-then-contain path stays authoritative.

Output: augmented bypass fallback + one mutation-proven fixture reproducing the
verifier's exact scenario + an extended in-place ADR-0009 marker disclosing the
fallback. Full suite stays 406 PASS / 0 FAIL / 1 SKIP (plus the new fixture).
