## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|----------------|
| operator's `AGENTS.md` → 0009's strip/insert awk | untrusted, hand-editable file content drives a state machine that rewrites the file in place and `mv`-commits the result | file content, in place |
| migration document → every target project | 0009's steps execute against repos this host does not own | migration steps, frontmatter |
| this repo's own version claim → every project it scaffolds | `SKILL.md`'s `version:` (installed globally) vs. `.codex/workflow-version.txt` (what the update skill reads) | version strings |
| the test suite → the migration | the suite is the only control asserting the migration is safe; a guard the suite cannot see fail is not covered | fixture assertions |
| pinned submodule (`vendor/agenticapps-shared`) → consumer | ships MECHANISM only; POLICY assertions must not be pushed across it (ADR-0035) | drift-test mechanism |
| upstream `claude-workflow` → this repo | ported code crosses a repo boundary; a moving HEAD makes a port unreproducible | ported fixture/regex logic |
| this repo → upstream's issue tracker | an outbound claim against another org's repo, made under the user's identity | vulnerability report |
| ADR-0010 → every future reader | a decision record carrying a falsified claim is trusted and re-applied | design rationale |
