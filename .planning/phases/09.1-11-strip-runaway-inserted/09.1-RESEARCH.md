*this scaffolder repo's own source tree*, which is exactly why the defect was invisible to
its author and why the suite manufactured a synthetic SKILL.md to keep it green. Do not
file V-01 upstream; upstream's version is correct for upstream's host.

The third finding is a correction to the code review's own remedy. **The reviewer's
preferred fix (a) is necessary but not sufficient**, and I reproduced the gap: un-latching

acknowledges producing at `:282`. Coupling entry to exit requires a terminal check as
well: `END { if (in_block && !swallowed_own_h2) exit 1 }`. The composite passes all four
legitimate shapes and both runaway shapes (§"Code Example 1").

**Primary recommendation:** Sequence as criterion 0 (V-01 — unblocks everything and is

### Pattern 1: Couple the strip's entry to its exit (criterion 2) — RECOMMENDED

**What:** Two changes, both required. Rule 3 (the reviewer's fix (a)) un-latches at any
structural boundary when the heading was never swallowed; the END guard fails closed when
no boundary ever arrives.

**When to use:** This is the criterion-2 fix. Rule 3 alone is **insufficient** — proven
below.

**Evidence that fix (a) alone is insufficient** [VERIFIED: executed this session]:

| Shape | fix (a) alone: in → out | Verdict |
|-------|--------------------------|---------|
| Drifted H2 **with** a later `## ` heading (the CR-01 repro shape) | 10 → 9 lines, all content preserved | ✅ closed |
| **Orphaned PROV, no heading, no later `## `** | **8 → 4 lines — still runs away to EOF** | ❌ **still open** |

The second shape is not hypothetical: `09-REVIEW.md` lists *"an orphaned provenance line
with no block at all"* as reachable, and `0009:282` documents that the migration **itself**
produces that state (*"the rest of the file, plus an orphaned provenance line"*). Rule 3
cannot fire because rule 3 needs a `## ` line to fire **on**.

**Verified composite** — see §"Code Example 1" for the full awk and the 8-shape result
table. All four legitimate shapes (fixture-06 block-to-EOF, healthy+`## `, fixture-09
two-provenance, marker-terminated) return rc=0 and correct output; both runaway shapes
return rc=1 (abort). ANCHOR-05's alternation is untouched.

**Two integration details the planner must not miss:**

1. **The abort message will lie.** The composite makes awk exit 1, which the shell's
   `if awk … > strip && [ -s strip ]; then` catches — routing to the existing `else` that
   prints *"the strip pass produced no output"*. That is false and misleading for this
   failure. The branch needs a distinguishable diagnostic (e.g. awk `exit 4` and a `case`
   on the status, or a sentinel line on stderr).
2. **`END { exit 1 }` and uninitialized-flag truthiness** are POSIX and verified on BSD awk
   20200816. `gawk`/`mawk` are **not installed on this machine** — portability is
   [ASSUMED] (POSIX-conformant), not verified. Same exposure the existing awk already
   carries.

## Common Pitfalls

### Pitfall 1: "Fixing" CR-01 by refusing on prose drift, contradicting D-26

**What goes wrong:** The natural fix ("if the block drifted, abort") collides with 0009's
own stated design (`0009:238-243`): *"The strip is deliberately BLIND to the block's
content … This migration must not refuse to PLACE a block just because its prose drifted —
content fidelity is 0004's job."*

**Why it happens:** "Heading drift" and "prose drift" get conflated.

**How to avoid:** They are different, and the mirror proves it. The canonical mirror has
**exactly one `## ` line, on line 1** — the heading (`grep -n '^## ' $MIRROR` → `1:` only,
[VERIFIED]). Everything below it is `### ` and prose. So:
- **Body/prose drift** → strip stays blind, block is re-vendored and repaired. D-26 holds.
- **Heading drift** → the block's *structural boundary marker* is gone. The strip cannot
  bound what it cannot find. Refusing is the only safe act.

**This is the reconciliation ADR-0010 should record for criterion 9**: D-26's principle
("content fidelity is 0004's job") survives intact; D-26's *claim* ("bounded by
construction, so a drifted block cannot cause a runaway") is false and must be corrected to
"bounded only when the heading matches — enforced by the entry/exit coupling."

**Warning sign:** A plan that adds a byte-comparison or a prose sentinel to the strip. That
is D-25's rejected content-sentinel pattern returning.

### Pitfall 2: Fixture numbering collides with upstream

**What goes wrong:** Criterion 6 names the new fixture `11-idempotent-rerun`. Upstream
`f9354cc` already ships **`11-prose-mention-provenance`** as *its* fixture 11 — and
criterion 3 asks this repo to port exactly that fixture. Two different fixture 11s.

**How to avoid:** Decide the numbering before writing either (see §"Open Questions" Q4).
Recommendation: mirror upstream (`11-prose-mention-provenance`) and give the idempotent
re-run `12-`, so the fixture-number correspondence with the repo 0009 is a port of stays
intact. This means **the ROADMAP's criterion 6 label is aliased, not satisfied literally**
— worth surfacing to the user.

**Warning sign:** A plan that says "add fixture 11" without saying which one.

### Pitfall 3: Making case 10(a) discriminate by exit code

**What goes wrong:** On a zero-byte mirror, guard 3 (`test -s`) **and** guard 4 (tail
sentinel `grep -q '^### 4\. Goal-Driven Execution$'`) both fail with `exit 3`. No exit-code
assertion can tell them apart — that is WR-04's structural point, and CR-03 is the same
fact seen twice.

**How to avoid:** Assert the **diagnostic**, which is the only thing that differs. Guard 3
prints *"vendored §11 canonical block missing or empty at:"* (`0009:112`). The planner
should confirm guard 4's message does not also contain that substring before locking the
`case` pattern.

**Warning sign:** A plan proposing to make 10(a) discriminate by "controlling the other
guard" — you cannot; guard 4 subsumes guard 3 on this input by construction.

### Pitfall 4: Editing the vendored submodule to satisfy criterion 8

**What goes wrong:** Criterion 8 says *"the drift test reads the version file."* The
obvious move is editing `run_drift_test`. That function lives in
`vendor/agenticapps-shared/migrations/lib/drift-test.sh` — a **git submodule** pinned at
`1f5d543` [VERIFIED: `.gitmodules` + `git ls-files -s`]. Editing it is a cross-repo change
that will not commit from here.

**How to avoid:** The lib's own header states the rule: *"This file ships the MECHANISM
only … the specific POLICY … is a consumer-owned invariant"* (ADR-0035). Criterion 8's
third leg is a **new consumer-side assertion in `test_drift()`** (`run-tests.sh:3198-3210`),
comparing `.codex/workflow-version.txt` against the SKILL.md version the mechanism already
reads. Do not touch `vendor/`.

**Warning sign:** A task whose file list includes anything under `vendor/`.

### Pitfall 5: Losing the RED by fixing and testing in one step

**What goes wrong:** Criteria 0 and 1 both require observing a FAILING fixture *before* the
corresponding fix. The suite is currently **314 PASS / 0 FAIL** — a green suite is the
phase's starting condition, and both new fixtures must break it.

**How to avoid:** Wave-separate RED from GREEN, exactly as Phase 9 did (09-03 RED gates
09-04 GREEN via `depends_on`). STATE.md's precedent is explicit: *"09-03 ends with a
failing suite by design. That RED is its deliverable (TEST-02), not a defect."*

**Warning sign:** A single plan that both adds the fixture and changes the awk.

### Pitfall 6: Trusting `grep -c` on BSD vs GNU

**What goes wrong:** The review's proposed CR-01 fix (b) uses `grep -c … | tr -d ' '` —
the `tr` is there because BSD `grep -c` output can carry leading whitespace in some
invocations. Fix (b) is not recommended (fix (a)+END is), but the same idiom appears
throughout `run-tests.sh` and any new counting assertion should follow it.

## Code Examples

### Example 1: The verified composite strip awk (criterion 2)

Verified this session against 8 shapes. `unresolved` catches a second PROV arriving while a
previous entry never found its heading; the END clause catches an entry that reaches EOF
unresolved. Rules are order-sensitive — the exact-heading swallow **must** precede the
un-latch rule.

```awk
# Source: composite of 09-REVIEW.md CR-01 fix (a) + an END fail-closed guard
#         (the reviewer's fix (a) alone leaves the orphaned-PROV shape open — verified)
BEGIN { in_block = 0; swallowed_own_h2 = 0 }

# ENTRY — anchored per CR-02 / upstream f9354cc:0029:223.
/^<!-- spec-source: agenticapps-workflow-core@[^[:space:]]+ §11 -->$/ {
  if (in_block && !swallowed_own_h2) { unresolved = 1 }   # prior entry never found its H2
  in_block = 1
  swallowed_own_h2 = 0
  next
}

# SWALLOW the block's own heading — MUST precede the un-latch rule below.
in_block && !swallowed_own_h2 && /^## Coding Discipline \(NON-NEGOTIABLE\)$/ {
  swallowed_own_h2 = 1
  next
}

# UN-LATCH (CR-01 fix (a)): provenance was not followed by the managed heading —
# do not consume the file. Print the boundary and resume.
in_block && !swallowed_own_h2 && (/^## / || /^<!-- gitnexus:start -->$/) {
  in_block = 0
  print
  next
}

# TERMINATOR (ANCHOR-05's alternation — unchanged, do not weaken).
in_block && swallowed_own_h2 && (/^## / || /^<!-- gitnexus:start -->$/) {
  in_block = 0
  swallowed_own_h2 = 0
  print
  next
}

in_block  { next }
!in_block { print }

# FAIL CLOSED: an entry that reached EOF without ever finding its heading ate the
# tail of the file. Refuse rather than commit the loss. A legitimate block that
# runs to EOF has swallowed_own_h2 == 1 and is unaffected (fixture 06).
END { if (unresolved || (in_block && !swallowed_own_h2)) exit 1 }
```

**Verified results** [VERIFIED: executed this session, BSD awk 20200816]:

| Shape | Expect | Got |
|-------|--------|-----|
| Orphan PROV, no later `## ` | abort | `rc=1` ✅ |
| Orphan PROV at EOF | abort | `rc=1` ✅ |
| Fixture-06: block runs to EOF | strip cleanly | `rc=0`, 4 lines ✅ |
| Healthy: block + later `## ` | strip cleanly | `rc=0`, 6 lines ✅ |
| Fixture-09: two-provenance-heal | strip both | `rc=0`, 9 lines ✅ |
| Block terminated by `gitnexus:start` | strip cleanly, marker survives | `rc=0`, 10 lines ✅ |
| Prose mention only, no real block | untouched | `rc=0`, **byte-identical** ✅ |
| CR-01 repro (drifted H2 + later `## `) | content survives | `rc=0`, 9 of 10 lines (only PROV dropped) ✅ |

The last row is worth the planner's attention: with the composite, a drifted-H2 file with a
following `## ` is **healed, not refused** — the drifted block's body stays and a fresh
canonical block is inserted at the anchor, so the operator sees a duplicate §11 rather than
data loss. That is a deliberate, defensible outcome (never destroy; surface the conflict)
but the planner should decide whether to state it in the ADR or tighten it. See Q1.

### Example 2: The falsifiable strip-integrity guard (criterion 4)

Must run **between** the passes, on the strip output, before the insert regenerates the
evidence. Adapted from the review's fix (b), which is a good *guard* even though fix (a)+END
is the better *fix*:

```bash
# Source: 09-REVIEW.md CR-01 fix (b), repositioned. Runs on AGENTS.md.0009.strip

  echo "       does not own (expected $(( h2_in - h2_own )) '## ' headings to"
```

   - What we know: the composite fix **heals** it — drifted body stays, canonical block is
     inserted at the anchor, operator sees two §11 sections. No data is lost. The orphan
     shapes refuse. Both behaviors verified.
   - What's unclear: the ROADMAP says only "cannot latch `in_block` to EOF" (criterion 2) —
     satisfied either way. D-26 says don't refuse on drift; the hand-paste guard says refuse
     on ambiguity. Duplication is arguably a worse operator experience than a clear abort.
   - Recommendation: **surface to the user before planning locks it.** Research leans
     "heal + duplicate" (never destroy, and D-26's principle is preserved), with a loud
     `echo` warning naming the drifted heading. If the user prefers refusal, add the
